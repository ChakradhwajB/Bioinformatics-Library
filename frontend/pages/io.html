Educational Guide: FASTA File Ingestion & JSON Exporters

1. Motivation: Why care?

In bioinformatics, storing and sharing genetic sequences is standardly performed using text-based file formats. The most common and universally supported format is FASTA. Without a standardized way to read and parse this raw sequence text, downstream computational analyses—like alignment, motif searching, and phylogenetic tree building—are impossible to automate.

2. Concept & Complexity

A FASTA file represents nucleotide or peptide sequences using single-letter codes. Each sequence begins with a single-line description (header), distinguished by a greater-than symbol (>). The sequence data follows on the subsequent lines.

  • Time Complexity: $\mathcal{O}(N)$ where $N$ is the total length of the file. The parser scans the file line-by-line exactly once.
  • Space Complexity: $\mathcal{O}(M)$ where $M$ is the size of the parsed sequences kept in memory as a dictionary structure.

3. Worked Example

Consider a file with the following contents: 

>seq1_human
ATGCGATA
>seq2_mouse
ATGC
GATA

The parser reads the first header seq1_human, then the sequence. For seq2_mouse, the sequence spans multiple lines, which the parser concatenates. The resulting mapped dictionary is:
{"seq1_human": "ATGCGATA", "seq2_mouse": "ATGCGATA"}

4. Python Code

Here is how to parse a FASTA file line-by-line and export the records as a JSON string in Python:

import json

def parse_fasta(file_content):
    records = {}
    current_header = None
    current_seq = []
    
    for line in file_content.splitlines():
        line = line.strip()
        if line.startswith(">"):
            if current_header:
                records[current_header] = "".join(current_seq)
            current_header = line[1:]
            current_seq = []
        else:
            current_seq.append(line)
            
    if current_header:
        records[current_header] = "".join(current_seq)
        
    return records

def export_to_json(records):
    return json.dumps(records, indent=4)

5. Practice Problems

Warm-up

Manually format a valid FASTA string containing two sequences of your choice. Ensure one sequence spans at least two lines.

Core

Modify the parse_fasta function to count and return the total number of sequences successfully parsed without using the len() function on the final dictionary.

Extension

FASTQ is another format that includes sequence quality scores. Research the FASTQ format and write a parser for it that extracts both the sequence and the quality string.
