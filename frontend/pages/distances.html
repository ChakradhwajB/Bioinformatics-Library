Educational Guide: Sequence Distance Metrics

1. Motivation: Why care?

When comparing two biological sequences, we need a mathematical way to quantify how "different" they are. Distance metrics allow us to build phylogenetic trees to track evolution or find the most likely origin of a mutated viral strain.

2. Concept & Complexity

We explore two primary metrics:

  • Hamming Distance: Counts only substitutions (point mutations) between two sequences of the same length. Time $\mathcal{O}(L)$, Space $\mathcal{O}(1)$.
  • Levenshtein (Edit) Distance: Counts the minimum number of insertions, deletions, and substitutions to transform one string into another. Handled via dynamic programming on a 2D grid. Time $\mathcal{O}(n \cdot m)$, Space $\mathcal{O}(n \cdot m)$.

3. Worked Example

Compare CAT vs CAR:

  • Hamming: 'C'=='C', 'A'=='A', 'T'!='R'. Distance = 1.
  • Levenshtein: Substitute 'T' for 'R'. Distance = 1.

Compare CAT vs CATS:

  • Hamming: Undefined (lengths differ).
  • Levenshtein: Insert 'S' at the end. Distance = 1.

4. Python Code

Here is how to compute both distance metrics in Python:

Hamming Distance:

def hamming_distance(seq1, seq2):
    if len(seq1) != len(seq2):
        raise ValueError("Lengths must match")
    return sum(1 for a, b in zip(seq1, seq2) if a != b)

Levenshtein Edit Distance:

def levenshtein_distance(seq1, seq2):
    n, m = len(seq1), len(seq2)
    dp = [[0 for _ in range(m + 1)] for _ in range(n + 1)]
    
    for i in range(n + 1): dp[i][0] = i
    for j in range(m + 1): dp[0][j] = j
        
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if seq1[i - 1] == seq2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + 1,       # Deletion
                    dp[i][j - 1] + 1,       # Insertion
                    dp[i - 1][j - 1] + 1   # Substitution
                )
                
    return dp[n][m]
