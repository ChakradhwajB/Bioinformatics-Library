Educational Guide: Overlapping K-mers

1. Motivation: Why care?

Comparing entire genomes base-by-base is computationally expensive and sometimes impossible if the sequences are fragmented. Instead, bioinformaticians break sequences into small, overlapping "words" of length $k$, called k-mers. By analyzing the frequency of these k-mers, we can estimate sequence similarity without alignment, identify repetitive elements, and piece together short DNA sequencing reads into full genomes using De Bruijn graphs.

2. Concept & Complexity

For a sequence $S$ of length $N$ over an alphabet $\Sigma$, the number of possible unique k-mers is bounded by $|\Sigma|^k$. The number of overlapping windows generated sequentially is $M = N - k + 1$.

The occurrence frequency of a specific k-mer pattern $P$ is mathematically expressed using the summation: $$\text{Freq}(P) = \sum_{i=0}^{N-k} I(S[i \dots i+k-1] == P)$$ where $I$ is an indicator function returning 1 if true, 0 otherwise.

  • Time Complexity: $\mathcal{O}(N \cdot k)$ — We slice $N - k + 1$ overlapping windows. Each string slice and hash takes $\mathcal{O}(k)$ time.
  • Space Complexity: $\mathcal{O}(N \cdot k)$ — In the worst case, every k-mer is unique, requiring memory proportional to the number of windows times their length.

3. Worked Example

Let's find all 3-mers ($k=3$) in the sequence $S = \text{"GATTACA"}$. We slide a window of size 3 from left to right:

GATTACA --- (GAT) --- (ATT) --- (TTA) --- (TAC) --- (ACA)

The resulting dictionary of frequencies is: {"GAT": 1, "ATT": 1, "TTA": 1, "TAC": 1, "ACA": 1}

4. Python Code

Here is how to extract overlapping k-mers using an optimized dictionary-based single-pass scanner:

def count_kmers(sequence, k):
    frequencies = {}
    n = len(sequence)
    
    # Slide a window of size k across the sequence
    for i in range(n - k + 1):
        kmer = sequence[i : i + k]
        if kmer in frequencies:
            frequencies[kmer] += 1
        else:
            frequencies[kmer] = 1
            
    return frequencies

5. Practice Problems

Warm-up

Given the sequence "ATATATA", what are all the unique 3-mers and their respective counts? Try to work it out by hand.

Core

Write a function find_most_frequent_kmers(seq, k) that builds on the counter above to return a list of the k-mer(s) that appear the most times in the sequence.

Extension
